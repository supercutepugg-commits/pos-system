# 커밋 컨벤션

## 형식

```
<prefix>: <요약>
- <상세 항목 1>
- <상세 항목 2>
```

- 모든 메시지는 **한글**로 작성
- 요약 및 상세 항목은 **명사형**으로 종결

## Prefix 종류

| Prefix   | 설명                  | 예시 요약                        |
| -------- | --------------------- | -------------------------------- |
| feat     | 새로운 기능 추가      | 로그인 기능 추가                 |
| fix      | 버그 수정             | 장바구니 수량 오류 수정          |
| refactor | 코드 리팩토링         | API 클라이언트 구조 개선         |
| docs     | 문서 추가/수정        | README 설치 가이드 추가          |
| style    | 코드 스타일/포맷팅    | import 순서 정리                 |
| chore    | 빌드/패키지/설정 변경 | pnpm 워크스페이스 설정           |
| test     | 테스트 코드           | 장바구니 Store 테스트 추가       |
| design   | UI/스타일 변경        | ProductCard 반응형 레이아웃 조정 |

## 규칙

1. 하나의 커밋은 하나의 논리적 단위만 포함
2. 여러 파일을 수정하더라도 동일한 목적이면 하나의 커밋으로 묶음
3. 서로 다른 목적의 변경은 반드시 별도 커밋으로 분리
4. 상세 항목은 파일명 또는 변경 내용을 간결하게 나열

첫 줄(`<prefix>: <요약>`) 형식은 커밋 시 `.husky/commit-msg` 훅으로 자동 검사됩니다.
